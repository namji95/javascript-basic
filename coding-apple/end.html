html : 글 넣고 그림 넣고 싶을 때 사용 css : 디자인 js : 기능

Library : 남들이 만들어둔 코드들을 뜻함 npm을 사용하여 라이브러리 관리가 쉬워짐 번들링 툴을 이용하여 안쓰는 코드를 삭제하여 필요한 용량 확보

npm : 패키지매니저 js 라이브러리 쉽게 설치 수정 삭제 가능 터미널에 npm install 원하는 라이브러리 명

js는 웹브라우저 안에서만 실행할 수 있는데 이걸 브라우저 안 켜고 바로 pc 안에서 js실행할 수 있게 해주는게 node.js

npm 설치 라이브러리는 node_modules에 있음 다른 파일을 가져다 쓰면 파일 구조가 복잡해지는데 bundle / build tool : 지금 필요한 자바스크립트 파일들을 전부 가져와 하나의 파일로 합쳐줌 자바스크립트 안에서 꼭 필요한 코드만 합쳐줌 .ts .vue .jsx 등으로 개발한 것들도 .js로 변환해줌 최신 문법을 호환성 좋게 바꿔줌 번들 : turbopack / vite / rollup.js / webpack / parcel

빌드란 번들링 툴을 이용해 파일을 하나로 합치고 싶으면 npm run build를 터미널에 작성 작성하면 폴더 생성 이 폴더 안에 내용들을 관리

vue / svelte / react / angular - CSR 자바스크립트로 html 조작을 편리하게 해주는 라이브러리 이것들을 이용하면 Single Page Application을 만들 수 있음 새로고침 없이 부드러운 사이트를 만들 수 있음 html 생성 변경이 쉬워져서 단점 변수들을 관리하기 힘듬 그래서 이 변수들을 관리하는 라이브러리가 있음 redux / recoil / mobx / zustand / jotai 더 큰 단점 1. 구글 검색 결과 노출이 어려움 2. 첫 페이지 로딩 시간이 오래걸림 3. 이것을 보완하기 위해서는 SSR 사용

리액트를 이용하여 SSR 문법 사용하고 싶다. 라이브러리 사용 next.js / nuxt.js / remix / sveltekit 자바스크립트(리액트) 문법으로 풀스택 개발가능 + SSR 가능

자바스크립트는 타입을 강제하는 기능이 없어서 타입으로 인한 버그 발생하는 경우 있음 Typescript 이걸 사용하면 됨 .ts 파일 만들어 사용하면 됨 번들링 툴 이용하여 js 파일로 변환하여 사용해야 한다.
